1 Economic Load Dispatch — Priority List

Ch6 · 5 คะแนน · เน้นคำนวณ + ตาราง Commitment

แนวคิดหลัก — เข้าใจก่อนจำสูตร

โหลดไฟฟ้าเปลี่ยนตลอดวัน — ชั่วโมงเที่ยงคืนโหลดต่ำ ชั่วโมงกลางวันโหลดสูง
Unit Commitment คือการตัดสินใจว่า ชั่วโมงไหนควรเปิด/ปิด Generator ตัวไหน เพื่อให้ต้นทุนรวมต่ำสุด
Priority List คือวิธีที่ง่ายที่สุด: จัดอันดับว่า Generator ตัวไหน "คุ้มค่า" ที่สุด แล้วเปิดตามลำดับ

สูตรหลักที่ต้องจำ

① Cost Function — ต้นทุนเชื้อเพลิง (฿/h)
$$F_i(P_i) = \alpha_i + \beta_i P_i + \gamma_i P_i^2$$

แทนค่า $P_i$ (MW) เข้าไป → ได้ต้นทุน ฿ ต่อชั่วโมง

② Priority Index σᵢ — เกณฑ์จัดลำดับ (฿/MWh)
$$\sigma_i = \frac{F_i(P_{\text{test}})}{P_{\text{test}}}$$

= ต้นทุนเฉลี่ยต่อ MWh ที่จุดทดสอบ
ยิ่ง σ น้อย = ยิ่งถูก = เปิดก่อน (Priority สูงกว่า)
$P_{\text{test}}$ อาจเป็น $P_{\max}$, $P_{\text{avg}}$, หรือ $P_{\min}$ — อ่านโจทย์ให้ดี (ส่วนใหญ่ใช้ $P_{\max}$)

③ Incremental Cost IC (ใช้ใน Lambda Iteration)
$$\text{IC}_i = \frac{dF_i}{dP_i} = \beta_i + 2\gamma_i P_i \quad [\text{฿/MWh}]$$ $$\lambda^* = \text{IC}_1 = \text{IC}_2 = \cdots \quad \Rightarrow \quad P_i^* = \frac{\lambda^* - \beta_i}{2\gamma_i}$$ $$\sum P_i^* = P_D \quad \text{(Power Balance)}$$

วิธีทำ Priority List ทีละขั้น

1
อ่านโจทย์: ใช้ $P_{\text{test}}$ แบบไหน? (Max / Avg / Min) → ใช้ค่านั้นตลอด
2
แทน $P_{\text{test}}$ ลงใน $F_i$ → ได้ต้นทุน ฿/h ของแต่ละ unit
3
คำนวณ $\sigma_i = F_i(P_{\text{test}}) \div P_{\text{test}}$ → ได้ ฿/MWh ของแต่ละ unit
4
เรียงลำดับ: $\sigma$ น้อย → Priority สูง → เปิดก่อน
5
สร้างตาราง Unit Commitment (ดูวิธีด้านล่าง — คนส่วนใหญ่งงตรงนี้)
6
(ถ้าโจทย์ถาม) Lambda Iteration: หา $P_i^*$ ของ units ที่ ON → ตรวจ limit → fix ถ้าเกิน

Step 6 — Lambda Iteration (Economic Dispatch)

หมายเหตุ: ตัวอย่าง Lambda Iteration ด้านล่างนี้เป็น เนื้อหาเพิ่มเติม ไม่ได้อยู่ในสไลด์อาจารย์โดยตรง (สไลด์ Page 42 มีแค่กราฟเปรียบเทียบ Priority List UC vs Economic Dispatch) — ใส่ไว้เผื่อข้อสอบให้คำนวณตัวเลข Economic Dispatch
ตัวอย่าง: Demand = 900 MW → State a (Unit 1+3+2 ทั้งหมด)

IC ของแต่ละ Unit

$$\text{IC}_1 = 7.2 + 2(0.00142)P_1 = 7.2 + 0.00284P_1$$ $$\text{IC}_2 = 7.85 + 2(0.00194)P_2 = 7.85 + 0.00388P_2$$ $$\text{IC}_3 = 7.97 + 2(0.00482)P_3 = 7.97 + 0.00964P_3$$

ตั้งสมการ IC₁ = IC₂ = IC₃ = λ → แสดง P ในรูป λ

$$P_1 = \frac{\lambda - 7.2}{0.00284}, \quad P_2 = \frac{\lambda - 7.85}{0.00388}, \quad P_3 = \frac{\lambda - 7.97}{0.00964}$$

แทนลง Power Balance: $P_1 + P_2 + P_3 = 900$

$$\frac{\lambda - 7.2}{0.00284} + \frac{\lambda - 7.85}{0.00388} + \frac{\lambda - 7.97}{0.00964} = 900$$ $$352.11(\lambda - 7.2) + 257.73(\lambda - 7.85) + 103.73(\lambda - 7.97) = 900$$ $$713.57\lambda - (2535.2 + 2023.2 + 827.0) = 900$$ $$713.57\lambda = 900 + 5385.4 = 6285.4$$
$\lambda^* \approx 8.81 \text{ ฿/MWh}$

หา P จาก λ*

$$P_1 = \frac{8.81 - 7.2}{0.00284} \approx 567 \text{ MW} \quad (150 \le 567 \le 600 \ \checkmark)$$ $$P_2 = \frac{8.81 - 7.85}{0.00388} \approx 247 \text{ MW} \quad (100 \le 247 \le 400 \ \checkmark)$$ $$P_3 = \frac{8.81 - 7.97}{0.00964} \approx 87 \text{ MW} \quad (50 \le 87 \le 200 \ \checkmark)$$
$567 + 247 + 87 \approx 901 \approx 900 \text{ MW} \quad \checkmark$
ไม่มี unit ไหน hit limit → ไม่ต้อง fix → เสร็จสิ้น

กรณีพิเศษ — ถ้า P* เกิน Limit

หลักการ: Fix unit ที่ hit limit แล้ว solve ใหม่
1
คำนวณ $P_i^* = (\lambda - \beta_i) / 2\gamma_i$ ตามปกติ
2
ถ้า $P_i^* > P_{i,\max}$ → Fix $P_i = P_{i,\max}$; ถ้า $P_i^* < P_{i,\min}$ → Fix $P_i = P_{i,\min}$
3
Demand ใหม่ = Demand เดิม − $P_{\text{fixed}}$ → แก้สมการใหม่สำหรับ units ที่เหลือ
4
ทำซ้ำจนไม่มี unit ไหน hit limit
ห้าม reuse ค่า $\lambda$ เดิมหลัง fix — ต้องแก้สมการหา $\lambda$ ใหม่ทุกครั้ง

2 AGC Multi-Area — ACE Calculation

Ch7 · 5 คะแนน · คำนวณ $\Delta P_{\text{tie}}$, ACE, บอกทิศทาง Gen

สูตรหลัก

Area Control Error (ACE)
$$\boxed{ACE_i = \Delta P_{\text{tie},i} + B_i \cdot \Delta f}$$

$\Delta P_{\text{tie},i} = P_{\text{tie,actual}} - P_{\text{tie,scheduled}}$ (Actual $-$ Schedule, มีเครื่องหมาย)
$B_i$ = Frequency Bias [MW/Hz] (กำหนดมาให้)
$\Delta f = f_{\text{actual}} - f_{\text{nominal}}$ [Hz]

ระบบ 2 Area — Tie-line Balance
$$\Delta P_{\text{tie},2} = -\Delta P_{\text{tie},1}$$

กำลังที่ส่งออกจาก Area 1 = กำลังที่รับเข้า Area 2 (เครื่องหมายตรงข้าม)

ระบบ 3 Area — Tie-line Balance Check
$$\sum_{i=A,B,C} \Delta P_{\text{tie},i} = 0 \quad \text{(ต้องเป็น 0 เสมอ)}$$

การแปลผล ACE

ค่า ACEความหมายAGC สั่ง
$ACE < 0$Area ผลิตน้อยไป / ส่งออกน้อยกว่า Scheduleเพิ่ม Generation ↑
$ACE > 0$Area ผลิตมากไป / ส่งออกมากกว่า Scheduleลด Generation ↓
$ACE = 0$สมดุลคงเดิม

ตัวอย่าง — ระบบ 2 Area (จาก Lecture)

โจทย์
  • $f = 49.95$ Hz, $f_{\text{nom}} = 50$ Hz $\;\Rightarrow\; \Delta f = 49.95 - 50 = -0.05$ Hz
  • Tie-line Schedule: Area 1 $\to$ Area 2 = $+300$ MW
  • Tie-line Actual: Area 1 $\to$ Area 2 = $+260$ MW
  • $B_1 = 400$ MW/Hz, $\;B_2 = 350$ MW/Hz
1
$\Delta P_{\text{tie}}$: $$\Delta P_{\text{tie},1} = 260 - 300 = -40\text{ MW}$$ $$\Delta P_{\text{tie},2} = -\Delta P_{\text{tie},1} = +40\text{ MW}$$
2
ACE: $$ACE_1 = (-40) + 400 \times (-0.05) = -40 - 20 = \mathbf{-60\text{ MW}} \quad\to\text{ เพิ่ม Gen ↑}$$ $$ACE_2 = (+40) + 350 \times (-0.05) = 40 - 17.5 = \mathbf{+22.5\text{ MW}} \quad\to\text{ ลด Gen ↓}$$

ตัวอย่าง — ระบบ 3 Area (จาก Lecture)

โจทย์
  • $f = 49.94$ Hz $\;\Rightarrow\; \Delta f = -0.06$ Hz
  • $B_A = 300$, $B_B = 250$, $B_C = 200$ MW/Hz
  • $\Delta P_{\text{tie},A} = -40$ MW, $\;\Delta P_{\text{tie},B} = +10$ MW, $\;\Delta P_{\text{tie},C} = +30$ MW
1
Check Balance: $$\sum \Delta P_{\text{tie}} = -40 + 10 + 30 = 0 \quad \checkmark$$
2
ACE แต่ละ Area: $$ACE_A = -40 + 300 \times (-0.06) = -40 - 18 = \mathbf{-58\text{ MW}} \quad\to\text{ เพิ่ม Gen ↑}$$ $$ACE_B = +10 + 250 \times (-0.06) = 10 - 15 = \mathbf{-5\text{ MW}} \quad\to\text{ เพิ่ม Gen ↑ (เล็กน้อย)}$$ $$ACE_C = +30 + 200 \times (-0.06) = 30 - 12 = \mathbf{+18\text{ MW}} \quad\to\text{ ลด Gen ↓}$$
สำคัญมาก: $\Delta P_{\text{tie},i}$ ต้องนิยามให้ถูก — เป็น Actual $-$ Schedule เสมอ และต้องมีเครื่องหมายแสดงทิศทาง
ในระบบ 3 Area ถ้า $\sum \Delta P_{\text{tie}} \ne 0$ แสดงว่าโจทย์กำหนด Scheduled หรือ Actual ผิดพลาด ต้องตรวจสอบก่อนคำนวณ ACE
$\Delta f < 0$ (ความถี่ต่ำกว่า nominal) มักทำให้ $B_i \cdot \Delta f < 0$ → ลด ACE → ดัน ACE เป็นลบ → เพิ่ม Gen

Derating Factor $C_s$ (IEEE 80)
$$C_s = 1 - \frac{1 - \dfrac{\rho_E}{\rho_s}}{1 + 22.22\, h_s}$$

$\rho_E$ = soil resistivity ($\approx 547 f^{-0.434} \approx 100$ Ω·m ที่ 50 Hz)
$\rho_s$ = surface material resistivity (Ω·m)
$h_s$ = ความหนาของวัสดุปูพื้น (หน่วยเมตร!)

ถ้าไม่มีวัสดุปูพื้น: $C_s = 1.0$ และ $\rho_s = \rho_E$ → ไม่ต้องคำนวณ $C_s$

ตัวอย่าง 3 กรณี (จาก Lecture — 70 kg, $\rho_E = 100$ Ω·m, $t_s = 1.0$ s)

กรณี ก — ไม่มีวัสดุปูพื้น
$$C_s = 1.0, \quad \rho_s = \rho_E = 100 \text{ Ω·m}$$ $$E_{\text{touch}} = (1000 + 1.5 \times 1.0 \times 100)\frac{0.157}{\sqrt{1.0}} = 1150 \times 0.157 = \mathbf{180.6\text{ V}}$$ $$E_{\text{step}} = (1000 + 6 \times 1.0 \times 100)\frac{0.157}{\sqrt{1.0}} = 1600 \times 0.157 = \mathbf{251.2\text{ V}}$$
กรณี ข — หินบดปูพื้น $h_s = 10\text{ cm} = 0.1\text{ m}$, $\rho_s = 3000$ Ω·m
$$C_s = 1 - \frac{1 - 100/3000}{1 + 22.22 \times 0.1} = 1 - \frac{0.9667}{3.222} = 1 - 0.300 = 0.700$$ $$E_{\text{touch}} = (1000 + 1.5 \times 0.700 \times 3000)\frac{0.157}{\sqrt{1.0}} = 4150 \times 0.157 = \mathbf{651.6\text{ V}}$$ $$E_{\text{step}} = (1000 + 6 \times 0.700 \times 3000)\frac{0.157}{\sqrt{1.0}} = 13600 \times 0.157 = \mathbf{2135.2\text{ V}}$$
กรณี ค — ยางมะตอยปูพื้น $h_s = 5\text{ cm} = 0.05\text{ m}$, $\rho_s = 10000$ Ω·m
$$C_s = 1 - \frac{1 - 100/10000}{1 + 22.22 \times 0.05} = 1 - \frac{0.990}{2.111} = 1 - 0.469 = 0.531$$ $$E_{\text{touch}} = (1000 + 1.5 \times 0.531 \times 10000)\frac{0.157}{\sqrt{1.0}} = 8965 \times 0.157 = \mathbf{1407.5\text{ V}}$$ $$E_{\text{step}} = (1000 + 6 \times 0.531 \times 10000)\frac{0.157}{\sqrt{1.0}} = 32860 \times 0.157 = \mathbf{5159.0\text{ V}}$$

5 Ground Grid Resistance — Sverak Formula

Ch8 · 5 คะแนน · อ่านรูปกริด + คำนวณ $R_G$

Sverak Formula (IEEE 80)

Sverak — Ground Grid Resistance
$$\boxed{R_G = \rho_E \left[\frac{1}{L_T} + \frac{1}{\sqrt{20A}}\left(1 + \frac{1}{1 + h\sqrt{20/A}}\right)\right] \quad [\Omega]}$$
Symbolความหมายหน่วยหมายเหตุ
$\rho_E$Soil ResistivityΩ·mกำหนดมาให้
$L_T$ความยาวรวมสายดินทั้งหมดmGrid + Ground Rods
$A$พื้นที่กริด = กว้าง × ยาวm²
$h$ความลึกฝัง (burial depth)m⚠ แปลง cm→m

วิธีอ่านรูปกริดเพื่อหา $L_T$

การนับเส้นจากรูป
$$L_T^{\text{grid}} = n_h \times W + n_v \times L$$ $$L_T^{\text{total}} = L_T^{\text{grid}} + N_{\text{rod}} \times L_{\text{rod}}$$

$n_h$ = จำนวนเส้นแนวนอน, $W$ = ความกว้างกริด (m)
$n_v$ = จำนวนเส้นแนวตั้ง, $L$ = ความยาวกริด (m)
ถ้า spacing เท่ากัน: $n = \text{จำนวนช่อง}+1$ (เช่น 10 ช่อง = 11 เส้น)

ตัวอย่างจาก Lecture (Slide 61–63)

โจทย์ — สถานีไฟฟ้า 22/115 kV
  • $\rho_E = 400$ Ω·m
  • กริด $70\text{ m} \times 70\text{ m}$ — 11 เส้นแนวนอน, 11 เส้นแนวตั้ง
  • Ground Rod 20 แท่ง × ยาว 7.5 m แต่ละแท่ง
  • $h = 0.5$ m (ความลึกฝัง)
1
คำนวณ $L_T$: $$L_T^{\text{grid}} = 11 \times 70 + 11 \times 70 = 770 + 770 = 1540\text{ m}$$ $$L_T^{\text{rods}} = 20 \times 7.5 = 150\text{ m}$$ $$L_T = 1540 + 150 = \mathbf{1690\text{ m}}$$
2
คำนวณ $A$: $$A = 70 \times 70 = 4900\text{ m}^2$$
3
คำนวณแต่ละ Term ในสูตร Sverak: $$\sqrt{20A} = \sqrt{20 \times 4900} = \sqrt{98000} = 313.05$$ $$\sqrt{\frac{20}{A}} = \sqrt{\frac{20}{4900}} = 0.06389$$ $$h\sqrt{\frac{20}{A}} = 0.5 \times 0.06389 = 0.03195$$ $$1 + h\sqrt{\frac{20}{A}} = 1.03195 \;\Rightarrow\; \frac{1}{1.03195} = 0.9691$$
4
แทนสูตร Sverak: $$R_G = 400\left[\frac{1}{1690} + \frac{1}{313.05}\left(1 + 0.9691\right)\right]$$ $$= 400\left[5.917 \times 10^{-4} + 3.194 \times 10^{-3} \times 1.969\right]$$ $$= 400 \times \left[5.917 \times 10^{-4} + 6.292 \times 10^{-3}\right]$$ $$= 400 \times 6.884 \times 10^{-3}$$
$R_G \approx 2.75\text{ Ω}$
เกณฑ์ IEEE — ค่า $R_G$ ที่ยอมรับได้

สถานีขนาดใหญ่ (transmission): $R_G \le 1\text{ Ω}$

สถานีขนาดเล็ก: $R_G$ อาจอยู่ที่ $1-5\text{ Ω}$

$h$ ต้องเป็นเมตร — ถ้าโจทย์ให้เป็น cm (เช่น 50 cm) ต้องเปลี่ยนเป็น 0.5 m ก่อน
นับ Ground Rod รวมใน $L_T$ ด้วย — อย่าลืม
นับเส้น = นับ conductor ไม่ใช่ช่อง: กริด 10 ช่อง → 11 เส้น

5 Steps — วิธีทำ

1
คำนวณ $I_{\text{load}}$ (primary) จาก MVA rating $$I_{\text{load}} = \frac{S_{\text{MVA}} \times 10^6}{\sqrt{3} \times V_{\text{kV}} \times 10^3} \quad [\text{A}]$$
2
เลือก CTR $$\text{CTR} = \frac{I_{\text{load}}}{5} \to \text{ปัดขึ้นเป็น Standard Ratio}$$ $$I_{\text{load,sec}} = \frac{I_{\text{load}}}{\text{CTR}}$$
3
เลือก Pickup $I_p$ (secondary) $$I_p = K \times I_{\text{load,sec}}, \quad K = 1.25 \sim 1.5$$ เลือก tap ที่ใกล้ที่สุดขึ้นไปจาก Standard Tap
ตรวจ Sensitivity: $PSM = I_{\text{fault,sec}} / I_p \ge 2$ (relay ต้องตอบสนอง fault ได้)
4
คำนวณ PSM $$PSM = \frac{I_{\text{fault,sec}}}{I_p} = \frac{I_{\text{fault,primary}} / \text{CTR}}{I_p}$$
5
คำนวณ $T$ หรือหา TMS
ถ้าโจทย์ให้ TMS มา → แทนสูตร หา $T$ ตรงๆ
ถ้าโจทย์ให้ $T$ ที่ต้องการ → แก้กลับหา TMS: $$\text{TMS} = T \times \frac{PSM-1}{13.5} \quad (\text{VI}), \quad \text{TMS} = T \times \frac{PSM^{0.02}-1}{0.14} \quad (\text{SI})$$

ตัวอย่างที่ 1 — Standard Inverse (SI) จาก Lecture

Feeder 1: 20 MVA, 115 kV, $I_{\text{fault}} = 2000$ A, SI Curve, TMS = 0.1

Step 1: $I_{\text{load}}$

$$I_{\text{load}} = \frac{20 \times 10^6}{\sqrt{3} \times 115 \times 10^3} = \frac{20\,000\,000}{199\,186} \approx 100\text{ A}$$

Step 2: เลือก CTR

โจทย์กำหนด CT = 600/5 → CTR = 120

$$I_{\text{load,sec}} = \frac{100}{120} \times 5 = 0.833 \times 5/5 \approx 0.83\text{ A (secondary)}$$

Step 3: $I_p$

$$I_{p,\text{primary}} = 1.25 \times 100 = 125\text{ A}$$

Step 4: PSM

$$PSM = \frac{I_{\text{fault}}}{I_{p,\text{primary}}} = \frac{2000}{125} = 16$$

Step 5: Trip Time (SI)

$$PSM^{0.02} = 16^{0.02} = e^{0.02\ln 16} = e^{0.0555} = 1.0570$$ $$T = 0.1 \times \frac{0.14}{1.0570-1} = \frac{0.014}{0.0570} = \mathbf{0.246\text{ s}} \approx 0.25\text{ s}$$
ผ่าน: Feeder ตัดที่ $\approx 0.25$ s ซึ่งเป็น fastest zone

ตัวอย่างที่ 2 — Very Inverse (VI) หา TMS เอง

$I_{\text{load}} = 200$ A, $I_{\text{fault}} = 2000$ A, VI Curve, ต้องการ $T = 0.4$ s

Step 2: เลือก CTR

$$\frac{200}{5} = 40 \to \text{เลือก CTR} = \frac{200}{5} = 40$$ $$I_{\text{load,sec}} = \frac{200}{40} = 5\text{ A}$$

Step 3: $I_p$

$$I_p = 1.25 \times 5 = 6.25\text{ A} \to \text{เลือก tap }125\% = 6.25\text{ A}$$

Step 4: PSM

$$I_{\text{fault,sec}} = \frac{2000}{40} = 50\text{ A}$$ $$PSM = \frac{50}{6.25} = 8.0$$ $$\text{Check: } \frac{50}{6.25} = 8 \ge 2 \quad\checkmark \text{ (Sensitive)}$$

Step 5: หา TMS จาก $T = 0.4$ s (VI)

$$T = \text{TMS} \times \frac{13.5}{PSM-1} \;\Rightarrow\; \text{TMS} = T \times \frac{PSM-1}{13.5} = 0.4 \times \frac{7}{13.5} = 0.207$$

เลือก TMS = 0.2 (ค่าใกล้ที่สุด)

ตรวจสอบ

$$T = 0.2 \times \frac{13.5}{8-1} = 0.2 \times 1.929 = \mathbf{0.386\text{ s} \approx 0.4\text{ s}} \quad\checkmark$$

Checklist ข้อสอบ

  • คำนวณ $I_{\text{load}}$ จาก $S/(\sqrt{3}V)$ ถูกหน่วย (VA, V)
  • เลือก CTR ปัดขึ้นเป็น Standard และ $I_{\text{load,sec}} \le 5$ A
  • เลือก $I_p$ tap ที่ใกล้ขึ้นไป (ไม่ต่ำกว่า $1.25 \times I_{\text{load,sec}}$)
  • ตรวจ Sensitivity: $PSM \ge 2$ สำหรับ fault current
  • จำสูตร SI/VI/EI/LTI ให้ถูกต้อง (ผิด 1 ตัวผลผิดหมด)
  • ถ้าโจทย์ให้ $T$ และถามหา TMS → แก้สมการกลับ
  • ระบุ TMS ให้อยู่ใน range 0.025–1.0
ผสมสูตร SI กับ VI ไม่ได้ — อ่านโจทย์ให้ดีว่าให้ใช้ curve ไหน (Standard Inverse / Very Inverse / Extremely Inverse)
